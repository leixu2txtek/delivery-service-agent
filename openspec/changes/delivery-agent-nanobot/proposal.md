## Why

现场实施人员经常遇到产品相关问题，当前依赖微信群和人工反馈，响应慢且知识分散。缺乏一个集中化的智能问答系统，导致实施效率低，尤其对缺乏经验的新人更为困难。需要构建一个基于 AI 的交付智能体，快速解答问题并引导问题闭环。

## What Changes

- **新增** 基于 nanobot 的智能问答系统，支持自然语言提问
- **新增** H5 移动端交互页面，支持实时对话
- **新增** 飞书知识库（MCP）集成，实时检索 FAQ 文档
- **新增** 引导诊断 Agent，通过预设流程分步收集信息并给出建议
- **新增** 转人工机制，自动发送飞书群消息并创建待办任务
- **新增** 用户认证系统，支持手机号+密码登录
- **新增** 历史会话管理，支持 7 天自动清理

## Capabilities

### New Capabilities
- `user-auth`: 用户认证能力，支持手机号+密码登录、JWT Token 管理、密码修改
- `faq-qa`: FAQ 智能问答能力，基于飞书 MCP 实时检索文档并生成答案
- `guided-diagnosis`: 引导诊断能力，通过预设流程分步收集信息并给出排查建议
- `escalation`: 转人工能力，自动发送飞书群消息并创建待办任务
- `conversation-mgmt`: 会话管理能力，支持历史记录查看和 7 天自动清理

### Modified Capabilities
- 无

## Impact

- **新系统**: nanobot 智能体服务、H5 前端应用、飞书 MCP 服务
- **新依赖**: OpenAI API（或兼容 OpenAI 协议的模型）、飞书开放平台、Redis、MySQL
- **基础设施**: 云服务器部署（阿里云/腾讯云/华为云）、域名、HTTPS 证书
- **用户群体**: 现场实施人员（需要培训使用）
