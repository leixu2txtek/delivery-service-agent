## ADDED Requirements

### Requirement: 用户可以自然语言提问并获得 FAQ 解答
系统 SHALL 接收用户自然语言问题，调用飞书 MCP 检索相关文档，并基于检索结果生成回答。

#### Scenario: 常见问题直接解答
- **WHEN** 用户提问"数据同步失败怎么处理"
- **THEN** 系统调用飞书 MCP 搜索相关文档
- **AND** 基于检索结果生成回答
- **AND** 回答中包含文档来源链接
- **AND** 返回时间不超过 5 秒

#### Scenario: 未找到相关内容
- **WHEN** 用户提问内容在飞书文档中无匹配
- **THEN** 系统返回"暂未找到相关内容，建议转人工咨询"
- **AND** 提供"转人工"按钮

#### Scenario: 多轮对话保持上下文
- **GIVEN** 用户已提问"数据同步失败"
- **WHEN** 用户追问"具体怎么检查 API 配置"
- **THEN** 系统理解追问上下文
- **AND** 基于前文和新的检索结果生成连贯回答

### Requirement: 系统 SHALL 展示回答来源
所有基于 FAQ 生成的回答 SHALL 附带参考文档链接，方便用户核实。

#### Scenario: 回答附带来源
- **WHEN** 系统返回 FAQ 回答
- **THEN** 回答底部显示"参考文档："
- **AND** 列出 1-3 个相关文档标题和链接

### Requirement: 系统支持流式输出回答
对于较长的回答，系统 SHALL 支持流式返回，提升用户体验。

#### Scenario: 流式回答
- **WHEN** 用户提问并建立 WebSocket 连接
- **THEN** 系统逐字/逐句返回回答内容
- **AND** 用户可实时看到内容生成
